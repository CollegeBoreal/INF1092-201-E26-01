Dans le cadre de l’atelier, nous avons procédé à la préparation et à l’installation du système d’exploitation sur un serveur physique.
Cette étape était essentielle avant de pouvoir automatiser la configuration via l’Infrastructure as Code.

🔧 1. Préparation du matériel
Avant toute manipulation logicielle, nous avons vérifié les composantes internes du serveur :

processeur

mémoire RAM

disques durs

alimentation

ventilation

Ensuite, nous avons raccordé les périphériques essentiels :

écran

clavier

souris

câbles d’alimentation

Cette étape permet de s’assurer que le serveur est fonctionnel avant l’installation.
2. Démarrage et accès au serveur
Après avoir mis le serveur sous tension :

Le système a démarré.

Nous avons accédé à l’interface de gestion.

Nous avons saisi les identifiants requis (nom d’utilisateur et mot de passe).

Une fois connectés, nous avons pu accéder aux paramètres du BIOS/UEFI.

💽 3. Configuration du boot
Nous avons configuré la machine pour qu’elle démarre sur la clé USB contenant l’installateur du système d’exploitation.

Cette étape est indispensable pour lancer l’installation depuis un support externe.
4. Documentation par captures d’écran
Les captures d’écran suivantes (placées dans le dossier images/) montrent :

le démarrage du serveur

l’accès au BIOS/UEFI

la sélection du périphérique USB

le lancement de l’installateur

Elles permettent de suivre visuellement chaque étape du processus.
Conclusion
Cet atelier m’a permis de :

comprendre les étapes de préparation d’un serveur

configurer un BIOS/UEFI pour booter sur un support externe

documenter correctement une installation

préparer le terrain pour l’utilisation de l’Infrastructure as Code
